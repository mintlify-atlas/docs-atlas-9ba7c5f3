## Step 1: Deploy with Docker

<Steps>
  <Step title="Run the Docker container">
    Execute this single command to start Memos:

    ```bash
    docker run -d \
      --name memos \
      -p 5230:5230 \
      -v ~/.memos:/var/opt/memos \
      neosmemo/memos:stable
    ```

    **What this does:**
    - `-d`: Run in detached mode (background)
    - `--name memos`: Name the container for easy management
    - `-p 5230:5230`: Expose Memos on port 5230
    - `-v ~/.memos:/var/opt/memos`: Persist data to `~/.memos` on your host
    - `neosmemo/memos:stable`: Use the stable release

    <Tip>
      **Data location**: All your memos, attachments, and database are stored in `~/.memos`. Back up this directory to preserve your data.
    </Tip>
  </Step>

  <Step title="Verify it's running">
    Check the container status:

    ```bash
    docker logs memos
    ```

    You should see output like:

    ```
    Memos 0.28.0 started successfully!
    Data directory: /var/opt/memos
    Database driver: sqlite
    Server running on port 5230
    Access your memos at: http://localhost:5230
    
    Happy note-taking!
    ```
  </Step>

  <Step title="Open Memos in your browser">
    Navigate to [http://localhost:5230](http://localhost:5230)

    You'll see the setup screen for creating your first user.
  </Step>
</Steps>

<Accordion title="Using Docker Compose instead?">
  Create a `docker-compose.yml` file:

  ```yaml docker-compose.yml
  version: '3'
  
  services:
    memos:
      image: neosmemo/memos:stable
      container_name: memos
      ports:
        - "5230:5230"
      volumes:
        - ~/.memos:/var/opt/memos
      restart: unless-stopped
      environment:
        # Optional: Set timezone
        - TZ=America/New_York
  ```

  Then run:
  ```bash
  docker-compose up -d
  ```
</Accordion>

## Step 2: Create Your Account

<Steps>
  <Step title="Sign up as the first user">
    The first user created automatically becomes an **admin** with full system permissions.

    Fill in the registration form:
    - **Username**: Your login username (alphanumeric, 4-32 characters)
    - **Password**: Strong password (min 8 characters recommended)
    - **Display Name**: How your name appears in memos (optional)

    <Warning>
      **Important**: After the first user is created, public registration is **disabled by default**. You can enable it later in Settings â†’ General â†’ Allow user registration.
    </Warning>
  </Step>

  <Step title="Sign in">
    After registration, you'll be automatically signed in and redirected to the home screen.

    <img src="/images/first-login.png" alt="Memos home screen" />
  </Step>
</Steps>

## Step 3: Create Your First Memo

<Steps>
  <Step title="Open the memo editor">
    At the top of the home screen, you'll see a text area with the placeholder: **"Any thoughts..."**

    Click to focus the editor. This is where you'll write your memos in Markdown.
  </Step>

  <Step title="Write your memo">
    Try this example to see Memos' features in action:

    ````markdown
    # Welcome to Memos! ðŸ‘‹

    This is my first memo. Let me test some features:

    ## Task List
    - [x] Install Memos
    - [ ] Write my first memo
    - [ ] Explore tags and attachments

    ## Tags
    I can use #tags to organize my thoughts. Tags like #productivity and #learning are automatically extracted.

    ## Code Blocks
    ```javascript
    // Syntax highlighting works great!
    console.log('Hello from Memos!');
    ```

    ## Links and Resources
    Check out the [documentation](https://usememos.com/docs) for more features.

    > **Note**: This memo is set to PRIVATE by default, so only I can see it.
    ````

    <Tip>
      **Markdown support**: Memos supports full GitHub-flavored Markdown including tables, footnotes, and even Mermaid diagrams!
    </Tip>
  </Step>

  <Step title="Set visibility">
    Before posting, choose who can see your memo:

    <Tabs>
      <Tab title="PRIVATE">
        **Default setting**
        - Only visible to you
        - Perfect for personal notes, drafts, and sensitive information
        - Most commonly used visibility setting
      </Tab>

      <Tab title="PROTECTED">
        - Visible to all logged-in users
        - Great for team knowledge sharing
        - Requires authentication to view
      </Tab>

      <Tab title="PUBLIC">
        - Visible to anyone with the link
        - No authentication required
        - Useful for public documentation or sharing specific notes

        <Warning>
          Your instance must have public visibility enabled (it's allowed by default).
        </Warning>
      </Tab>
    </Tabs>
  </Step>

  <Step title="Post your memo">
    Click the **"POST"** button or press `Ctrl+Enter` (or `Cmd+Enter` on Mac).

    Your memo appears in the timeline below! ðŸŽ‰
  </Step>
</Steps>

## Advanced Features

<AccordionGroup>
  <Accordion title="Memo Relations" icon="link">

    2. Click tasks to toggle completion
    3. View all tasks in sidebar â†’ **Tasks** view
    4. Filter by incomplete/complete

    ```markdown

    - [x] Review PRs
    - [ ] Write documentation
    - [ ] Deploy to production
    ```

  <Accordion title="Mermaid Diagrams" icon="diagram-project">
    Create flowcharts, sequence diagrams, and more:


  <Accordion title="RSS Feeds" icon="rss">

## Common Configuration

<Steps>
  <Step title="Change default visibility">
    Navigate to: **Settings** â†’ **Preferences** â†’ **Default memo visibility**

    Choose PRIVATE, PROTECTED, or PUBLIC as your default when creating memos.
  </Step>

  <Step title="Enable user registration">
    As an admin: **Settings** â†’ **General** â†’ **Allow user registration**

    Enable this to let others create accounts on your instance.
  </Step>

  <Step title="Configure storage">
    **Settings** â†’ **Storage** â†’ Choose between:
    - **Local storage** (default): Files stored in data directory
    - **S3-compatible**: Use AWS S3, MinIO, Cloudflare R2, etc.

    ```bash
    # Configure S3 via environment variables
    docker run -d \
      --name memos \
      -p 5230:5230 \
      -v ~/.memos:/var/opt/memos \
      -e MEMOS_DRIVER=postgres \
      -e MEMOS_DSN="postgres://user:pass@localhost/memos" \
      neosmemo/memos:stable
    ```
  </Step>
</Steps>

## Backup Your Data

<Warning>
  **Important**: Always backup your data directory before upgrades!
</Warning>

```bash
# Stop the container
docker stop memos

# Backup data directory
tar -czf memos-backup-$(date +%Y%m%d).tar.gz ~/.memos/

# Restart container
docker start memos
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Can't access http://localhost:5230">
    **Check if container is running:**
    ```bash
    docker ps | grep memos
    ```

    **Check logs for errors:**
    ```bash
    docker logs memos
    ```

    **Port already in use?** Change the host port:
    ```bash
    docker run -d --name memos -p 8080:5230 -v ~/.memos:/var/opt/memos neosmemo/memos:stable
    ```
    Then access at `http://localhost:8080`
  </Accordion>

  <Accordion title="Lost admin password">
    Reset password via command line:

    ```bash
    docker exec -it memos /usr/local/memos/memos \
      --data /var/opt/memos \
      --admin-password-reset
    ```
  </Accordion>

  <Accordion title="Permission errors">
    Ensure the data directory is writable:

    ```bash
    mkdir -p ~/.memos
    chmod 755 ~/.memos
    ```
  </Accordion>

  <Accordion title="Database migration failed">
    If upgrading from old version:

    1. Backup your data first!
    2. Check minimum version requirement in release notes
    3. Memos requires version 0.22+ for migrations

    See [Upgrade Guide](#upgrading) for details.
  </Accordion>
</AccordionGroup>
