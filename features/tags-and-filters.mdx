Memos provides flexible organization through hashtags and advanced filtering powered by Common Expression Language (CEL).

## Using Tags

### Creating Tags

Tags are created automatically by typing `#` followed by text:

```markdown
Working on the #backend API today.
This relates to #project-alpha and #development.

Meeting with #team tomorrow to discuss #roadmap.
```

**Tag syntax:**
- Start with `#` symbol
- Alphanumeric characters (a-z, A-Z, 0-9)
- Hyphens and underscores allowed
- Unicode characters supported (Chinese, Japanese, Korean, etc.)
- Case-sensitive: `#Work` and `#work` are different
- No spaces (use hyphens: `#team-meeting`)

### Tag Behavior

<Tabs>
  <Tab title="Case Sensitivity">
    Tags preserve exact case:
    
    ```markdown
    #Work #work #WORK
    ```
    
    These create **three different tags**. For consistency:
    - Choose a convention (lowercase, Title Case, etc.)
    - Stick to it across all memos
    - Use tag autocomplete to select existing tags
  </Tab>
  
  <Tab title="Deduplication">
    Duplicate tags in a single memo are removed:
    
    ```markdown
    Important #meeting today. Don't forget the #meeting!
    ```
    
    Results in tags: `["meeting"]` (appears once)
  </Tab>
  
  <Tab title="Autocomplete">
    The editor suggests existing tags:
    
    1. Type `#` in the editor
    2. Start typing tag name
    3. See matching existing tags
    4. Press `Tab` or `Enter` to select
    5. Maintains consistency automatically
  </Tab>
</Tabs>

## Filtering Memos

### Tag Filters

Click any tag to filter memos:

1. Click a `#tag` in any memo
2. View all memos with that tag
3. Click the tag filter to remove it
4. Combine with other filters

Or use the search:

```
Search for: #project-alpha
Results: All memos tagged with #project-alpha
```

## Advanced Filtering with CEL

Memos uses [Common Expression Language](https://github.com/google/cel-spec) for powerful filtering.

### Basic Syntax

```cel
# Equality
tag_search == "work"
visibility == "PRIVATE"

# Comparison
create_time > timestamp("2024-01-01T00:00:00Z")

# String operations
content_search.contains("meeting")

# Boolean logic
property.has_code == true

# Negation
!(visibility == "PUBLIC")
```

### Combining Filters

Use logical operators:

```cel
# AND - both conditions must be true
tag_search == "work" && property.has_task_list == true

# OR - either condition can be true
tag_search == "urgent" || property.has_incomplete_tasks == true

# Complex combinations
(tag_search == "work" || tag_search == "project") && 
create_time > timestamp("2024-01-01T00:00:00Z")
```

### Field Reference

| Field | Type | Description | Example |
|-------|------|-------------|----------|
| `tag_search` | string | Filter by tag | `tag_search == "work"` |
| `content_search` | string | Search content | `content_search.contains("API")` |
| `visibility` | string | Memo visibility | `visibility == "PRIVATE"` |
| `pinned` | bool | Pinned status | `pinned == true` |
| `creator_id` | int | Creator user ID | `creator_id == 123` |
| `create_time` | timestamp | Creation time | `create_time > timestamp("2024-01-01T00:00:00Z")` |
| `update_time` | timestamp | Last update | `update_time > create_time` |
| `display_time` | timestamp | Display time | `display_time.getDate() == 1` |
| `property.has_link` | bool | Contains links | `property.has_link == true` |
| `property.has_code` | bool | Contains code | `property.has_code == true` |
| `property.has_task_list` | bool | Contains tasks | `property.has_task_list == true` |
| `property.has_incomplete_tasks` | bool | Has unchecked tasks | `property.has_incomplete_tasks == true` |

### Date and Time Filters

```cel
# Memos from 2024
create_time >= timestamp("2024-01-01T00:00:00Z") &&
create_time < timestamp("2025-01-01T00:00:00Z")

# Last 7 days (requires relative timestamps in implementation)
create_time > timestamp("2024-01-20T00:00:00Z")

# Specific date
display_time.getDate() == 15

# Specific month
display_time.getMonth() == 1  # January (0-indexed)

# Weekends (Saturday=6, Sunday=0)
display_time.getDayOfWeek() == 0 || display_time.getDayOfWeek() == 6
```

## Shortcuts

Save frequently used filters as shortcuts for quick access.

### Creating Shortcuts

1. Click **Create Shortcut** in the sidebar
2. Enter a **title** (e.g., "Work TODOs")
3. Enter your **filter expression**
4. Save

<CodeGroup>
```cel Work TODOs
tag_search == "work" && property.has_incomplete_tasks == true
```

```cel Recent Code
property.has_code == true && 
create_time > timestamp("2024-01-01T00:00:00Z")
```

```cel Team Updates
tag_search == "team" || tag_search == "update"
```
</CodeGroup>

### Using Shortcuts

1. Click a shortcut in the sidebar
2. Memos are filtered automatically
3. Click again to clear the filter

## Organizing Tags

### Tag Hierarchies

Use slashes for nested organization:

```markdown
#work/project/backend
#work/project/frontend
#work/admin/hr
#work/admin/finance

#book/fiction/scifi
#book/fiction/fantasy
#book/nonfiction/history
```

**Benefits:**
- Logical grouping
- Easy filtering at any level
- Clear relationships
- Scalable organization

**Filtering hierarchies:**
```cel
# All work-related
tag_search.startsWith("work/")

# Specific project
tag_search == "work/project/backend"

# All project tags
tag_search.startsWith("work/project/")
```

## API Integration

### Filter Validation

Validate filters before saving:

```javascript
try {
  await client.createShortcut({
    parent: "users/123",
    shortcut: {
      title: "Test",
      filter: 'tag_search == "work"',
    },
    validateOnly: true, // Only validate, don't create
  });
  console.log("Filter is valid!");
} catch (error) {
  console.error("Invalid filter:", error.message);
}
```

### List Memos with Filters

```javascript
const response = await client.listMemos({
  filter: 'tag_search == "work" && property.has_incomplete_tasks == true',
  orderBy: "pinned desc, display_time desc",
  pageSize: 50,
});

for (const memo of response.memos) {
  console.log(`${memo.content} - ${memo.tags.join(", ")}`);
}
```

### Extract Tags

Tags are automatically extracted:

```javascript
const memo = await client.createMemo({
  memo: {
    content: "Working on #project with #team",
  },
});

console.log(memo.tags); // ["project", "team"]
```
